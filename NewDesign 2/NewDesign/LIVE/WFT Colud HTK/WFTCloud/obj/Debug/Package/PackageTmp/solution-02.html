SAP Disaster Recovery as a Service (DRaaS)

WFT Cloud’s SAP Disaster Recovery as a Service(DRaaS) is an affordable, simple and enterprise class disaster recovery solution for SAP landscapes. Disaster recovery-as-a-service solutions allow customers to avoid the costs (CAPEX) of purchasing the infrastructure and software needed for a secondary disaster recovery site and allow to pay for your disaster recovery solution out of your operating budget (OPEX).

WFT Cloud’s SAP Disaster Recovery as a Service(DRaaS) offers a comprehensive set of tools and services to assess, design and build disaster recovery environment for SAP in cloud. WFT Cloud DRaaS solutions support Cloud provider agnostic DRaaS models, this enables the customers to choose from a variety of SAP certified cloud service providers including a combination of Public (AWS, Azure), Private or Hybrid cloud.

WFT is an SAP-certified provider of cloud and hosting services. Certified professionals in WFT’s Global SAP Center of Excellence(CoE) provide expertise in helping enterprise customers to design and build DRaaS solution as per their recovery time objective (RTO) and recovery point objective (RPO). WFT Cloud also will help the customers to test the DR solution when required.

Key Features

Assess

Evaluate the customer existing SAP landscapes understand current SAP landscape characteristics, customer RPO and RTO requirements, compliance requirements, testing requirements and budget.

Design

Design DRaaS cloud options that meet the customer RPO and RTO requirements with a single provider or combination of public (AWS, Azure), private or hybrid cloud deployment models.

Build

Build the Disaster Recovery environment conforming to the requirements and design. This include setting up replication (including pre-seeding of data), building target DR environment and establishing network connectivity between primary and DR site(s). As part of the build process WFT will also develop a DR runbook for customers for execution of DR plan in the event of a disaster.

Test

WFT will work with customer to carry out DR failover tests as required.

Operate

WFT will manage the DR environment, this includes monitoring and replicating any configuration changes incurred in the source site to DR site.
